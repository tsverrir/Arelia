# Known Issues & Gotchas

This document records issues, bugs, edge cases, and developer gotchas discovered during implementation and testing.
Use the template below when registering a new issue. Increment the issue number sequentially.

---

## ISSUE-000 — Template (example — delete when adding a real issue)

**Severity:** Low | Medium | High | Critical  
**Status:** Open | 🔍 Investigating | ✅ Fixed  
**Area:** Layer / Component (e.g. `Application / InviteUser.cs`, `Web / RoleDetail.razor`)

**Description:**  
One or two sentences describing what is wrong and under what conditions it occurs.

**Impact:**  
What goes wrong for the user or developer. Include workarounds if any exist.

**Fix:**  
Steps taken or proposed to resolve the issue (replace with **Fix applied:** once done).
